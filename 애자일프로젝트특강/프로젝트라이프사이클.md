# 프로젝트라이프사이클

### 적응형 라이프 사이클 개요

- 예측형이든 적응형이든 모든 프로젝트는 프로세스가 존재함
- 표준 프로세스
  - 분석
  - 디자인
  - 개발
  - 테스트
- 적응형은 예측형보다 주기가 더 짧음



### 반복형 라이프 사이클

반복(Iteration)이란? : 일련의 반복적인 주기를 통해 제품을 개발하는 것을 의미

#### 특징

- 프로젝트 초기에 프로젝트 요구사항을 수집하고 범위를 결정함
- 프로토타입을 통해서 분석과 설계를 반복함
- 개발과 테스트를 여러 번 반복 수행하여 제품이나 서비스를 구체화하고 개선함
- 반복을 통해서 요구사항이 충족되면, 마지막에 한번 인도함
- 프로젝트 팀의 제품 이해도가 높아지면, 일상적으로 프로젝트의 시간과 원가 산정치를 수정함



#### 반복형 라이프사이클이 필요한 경우

- 프로젝트가 상대적으로 복잡한 경우
- 잦은 변경이 발생하는 경우
- 프로젝트 팀과 이해관계자가 프로젝트 범위에 대해서 기대하는 것이 다를 경우



### 프로토 타입

#### 프로토타입 

- 프로토타입 = 시제품 = 작동 모형 = 예상 제품의 핵심 요구사항을 갖춘 시범적 모델

- 고객의 요구사항을 가장 정확하게 확인할 수 있음

  - 프로토타입 제작은 실제 결과물을 만들기 전에 예상 제품의 핵심 요구사항을 갖춘 시범적인 모델을 만들어 제공함으로써 요구사항에 대한 조기 피드백을 확보할 수 있는 방법임

  - 프로토타입은 유형의 모형이므로 이해관계자가 요구사항 요약서를 놓고 토론하는데 그치지 않고 최종 제품의 모형으로 실험할 수 있는 기회를 제공함 -> 공격적 정교함의 개념을 지원함



### 증분형 라이프 사이클 

= 점증적 라이프 사이클

#### 특징

- 증분형 라이프 사이클에서는 사전에 정해진 기간 내에 기능을 계속 추가해 나가는 일련의 반복 과정을 통해 인도물이 산출됨
- 프로젝트 인도물에는 역량도 포함되어야 하며, 이러한 조치는 일반적으로 최종 반복 이후에 진행됨 = 최종 반복 단계 이후에만 완성된 것으로 간주되는 필요충분 역량이 인도물에 포함됨
